# Documents Directory

This directory stores generated specification documents in the document pipeline.

## Structure

```
documents/{project_id}/
├── prd.md    # Product Requirements Document
├── srs.md    # Software Requirements Specification
└── sds.md    # Software Design Specification
```

## Document Flow

```
Collector → PRD Writer → SRS Writer → SDS Writer
              ↓             ↓             ↓
           prd.md        srs.md        sds.md
```

## File Descriptions

### prd.md
Product Requirements Document generated by PRD Writer (CMP-002):
- Problem statement
- Solution overview
- User personas
- Functional requirements
- Non-functional requirements
- Constraints and assumptions

### srs.md
Software Requirements Specification generated by SRS Writer (CMP-003):
- Feature specifications (SF-XXX)
- Use cases (UC-XXX)
- Acceptance criteria (AC-XXX)
- System requirements
- Interface requirements

### sds.md
Software Design Specification generated by SDS Writer (CMP-004):
- Architecture design
- Component design (CMP-XXX)
- Data design
- Interface design
- Security design

## Approval Flow

Each document may require user approval before the next stage proceeds (configurable in workflow.yaml).

## Notes

- Documents use the templates from `.ad-sdlc/templates/`
- Versioning is tracked via Git commits
- Korean versions (*.kr.md) may also be generated if configured
